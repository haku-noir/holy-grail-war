# CPUアルゴリズム仕様書

本ドキュメントでは、「聖杯戦争」ゲームにおけるCPU対戦相手の思考アルゴリズムについて解説します。
難易度はレベル1からレベル3まで用意されています。

## レベル1: Random (ランダム)
- **概要**: 完全にランダムに行動します。
- **ロジック**:
  - 現在の手札からランダムに1枚を選択してプレイします。
  - 戦略的な思考は一切行いません。

## レベル2: Standard (ターン最適)
- **概要**: その場その場で、最も勝率の高い手を選びます。
- **ロジック**:
  1. 自分の手札にあるカード候補それぞれについて、シミュレーションを行います。
  2. 相手の現在の手札にある全てのカードと対戦させます（総当たり）。
  3. 相手が手札からランダムにカードを出すと仮定し、勝利数（引き分けは0.5勝）をカウントします。
  4. 最も勝利数が多い（期待値が高い）カードを選択します。
- **特徴**: 短期的な勝利を目指しますが、先の展開（次のターンのバフや聖杯の数など）は考慮しません。

## レベル3: Ultimate (ゲーム最適)
- **概要**: ゲーム終了までの全展開を読み切り、最終的な勝利（聖杯獲得数 > 武勲）を目指します。
- **ロジック**:
  - **マキシミン戦略（Maximin Strategy）**を採用したゲーム木探索を行います。
  - 自分と相手が同時にカードを出すゲームですが、思考モデルとしては「自分がカードを出した後、相手がそれに対する最善手（自分にとっての最悪手）を出してくる」という最悪ケースを想定します。
  - **探索**: 残りのターン数が少ない（最大5ターン）ため、ゲーム終了まで完全探索を行います。
  - **評価関数**:
    - ゲーム終了時の聖杯数を比較し、勝利なら高スコア、敗北なら低スコアを与えます。
    - 聖杯数が同数の場合は、獲得したカードの合計パワー（武勲）で判定します。
  - 現在の手札の各候補について「最悪ケースのスコア（相手が最適に応手した場合のスコア）」を計算し、その中で最もマシな（スコアが高い）手を選択します。

### 補足: 完全情報と非完全情報
- 現在の実装では、CPUは「相手の手札の内容」を完全に把握している前提で動作します（チートAI）。
- プレイヤーが「このターンに何を出したか」は知りません（同時出しルールは守られます）。
